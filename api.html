Documentation API REST Pokémon

1. Pokédex

Route de recherche

/api/pokedex/search?generation=<num>&categorie=<nom>&nom=<syllabe>&langue=<fr|jp|en>&type=<type>&talent=<nom>

Description

Cette route permet de rechercher des Pokémon en utilisant différents critères. Tous les paramètres sont optionnels et peuvent être combinés pour affiner la recherche.

Paramètres

Paramètre Description Exemple
generation Numéro de la génération du Pokémon 1, 2, 3, etc.
categorie Catégorie du Pokémon starter, légendaire, etc.
nom Syllabe ou partie du nom du Pokémon pika, sala, etc.
langue Langue du nom du Pokémon fr, jp, en
type Type du Pokémon feu, eau, électrique, etc.
talent Nom du talent du Pokémon statik, intimidation, etc.

Autres fonctionnalités

Statistiques globales

/api/pokedex/stats/global

Renvoie des statistiques globales sur tous les Pokémon.

Comparaison de statistiques

/api/pokedex/stats/comparer?pokemon=<nom1>,<nom2>,...

Compare les statistiques de plusieurs Pokémon.

Résistances globales

/api/pokedex/resistances/global

Renvoie des informations sur les résistances de tous les types de Pokémon.

Comparaison de résistances

/api/pokedex/resistances/comparer?pokemon=<nom1>,<nom2>,...

Compare les résistances de plusieurs Pokémon.

Résistances par élément

/api/pokedex/resistances/element/<type>?multiplicateur=<valeur>

Recherche les Pokémon avec une résistance spécifique à un type d'attaque.

Chaîne d'évolution par nom

/api/pokedex/evolution/nom/<nom>

Renvoie la chaîne d'évolution d'un Pokémon recherché par son nom.

Chaîne d'évolution par ID

/api/pokedex/evolution/id/<id>

Renvoie la chaîne d'évolution d'un Pokémon recherché par son ID.

Caractéristiques spécifiques

/api/pokedex/caracteristiques/<nom>?inclure=taille,poids,reproduction,ratio_genre

Renvoie des caractéristiques spécifiques d'un Pokémon. Les caractéristiques à inclure sont optionnelles.

Exemples

/api/pokedex/search?generation=1&categorie=starter&nom=sa&langue=fr&type=feu
/api/pokedex/recherche?generation=4
/api/pokedex/recherche?nom=pika&langue=jp
/api/pokedex/stats/comparer?pokemon=pikachu,raichu,bulbizarre
/api/pokedex/resistances/element/feu?multiplicateur=0.5
/api/pokedex/evolution/nom/evoli
/api/pokedex/caracteristiques/dracaufeu?inclure=taille,poids,ratio_genre

2. Items

Route de recherche

/api/items/recherche?nom=<syllabe>&generation=<num>&apparition=<syllabe>&serie=<nom>&rangement=<type>&achat=<prix>&revente=<prix>

Description

Cette route permet de rechercher des items en utilisant différents critères. Tous les paramètres sont optionnels et peuvent être combinés pour affiner la recherche.

Paramètres

Paramètre Description Exemple
nom Syllabe ou partie du nom de l'item pok, ball, etc.
generation Numéro de la génération où l'item est apparu 1, 2, 3, etc.
apparition Lieu d'apparition de l'item kanto, johto, etc.
serie Série de jeux où l'item est présent xy, sm, etc.
rangement Type de rangement de l'item sac, boîte, etc.
achat Prix d'achat de l'item 100, 1000, etc.
revente Prix de revente de l'item 50, 500, etc.

Exemples

/api/items/recherche?nom=pok&generation=1&apparition=kanto
/api/items/recherche?nom=ball
/api/items/recherche?generation=3
/api/items/recherche?serie=xy&rangement=sac&page=2&limite=20&tri=nom&ordre=asc

3. Attaques de Pokémon

Route de recherche

/api/attaques_pokemon/recherche?attaque=<nom>&generation=<num>&puissance=<valeur>&pp=<valeur>

Description

Cette route permet de rechercher des attaques de Pokémon en utilisant différents critères. Tous les paramètres sont optionnels et peuvent être combinés pour affiner la recherche.

Paramètres

Paramètre Description Exemple
attaque Nom ou partie du nom de l'attaque lance, éclair, etc.
generation Numéro de la génération où l'attaque est apparue 1, 2, 3, etc.
puissance Puissance de l'attaque 40, 90, 100, etc.
pp Points de pouvoir (PP) de l'attaque 5, 10, 15, etc.

Exemples

/api/attaques_pokemon/recherche?attaque=lance&generation=1&puissance=90
/api/attaques_pokemon/recherche?attaque=surf
/api/attaques_pokemon/recherche?puissance=100

4. Pagination et Tri

Paramètres

&page=<numéro>&limite=<nombre>&tri=<champ>&ordre=<asc|desc>

Description

Ces paramètres peuvent être ajoutés à n'importe quelle route de recherche pour paginer et trier les résultats.

Paramètres

Paramètre Description Exemple
page Numéro de la page à afficher 1, 2, 3, etc.
limite Nombre d'éléments par page 10, 20, 50, etc.
tri Champ sur lequel effectuer le tri nom, puissance, etc.
ordre Ordre de tri (ascendant ou descendant) asc, desc

Exemple

/api/items/recherche?nom=pok&page=2&limite=20&tri=nom&ordre=asc
